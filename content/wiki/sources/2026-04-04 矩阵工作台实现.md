# 2026-04-04 矩阵工作台实现计划

本文档详细记录了将现有验收表单系统重构为"A 方案矩阵工作台”的完整实施计划。重构的核心目标是将旧的“多次数测试方块”模型转变为基于动态蛇形矩阵和稀疏异常记录的高效工作流。

## 核心目标

- **架构转型**：从固定网格全量记录转向动态蛇形矩阵与稀疏存储。
- **交互升级**：采用左侧固定 `2x2` 四宫格（PointInspector）与右侧密集矩阵总览（PointGrid）的双栏布局。
- **数据优化**：实施“默认成功”策略，数据库中仅存储失败点和备注，显著减少数据量。
- **开发策略**：严格遵循测试驱动开发（TDD）流程，确保重构过程中的代码质量。

## 技术栈

- **前端/壳子**: VitePress, Vue 3, TypeScript
- **后端/数据**: Node.js, SQLite
- **测试**: Vitest
- **工具**: XLSX (数据导出)

## 关键实施任务

### 1. 动态矩阵与蛇形映射
实现 `src/shared/matrix.ts` 模块，负责根据总点数动态计算行列数，并提供蛇形编号（Snake-like Mapping）与坐标的双向转换算法。这是新视觉布局的数学基础。

### 2. 稀疏记录模型
修改数据类型定义与统计逻辑，确立“缺失即成功”的语义。仅在数据库中记录 `isFailed: true` 或包含备注的点位，优化查询与存储效率。

### 3. 数据库与仓储层重构
废弃旧的 `scene_results` 表，新建 `scene_point_records` 表。重写仓储层以支持稀疏数据的读写、状态切换及新格式的 Excel 导出。

### 4. 前端交互重建
- **PointInspector**: 实现左侧四宫格，展示当前点及其相邻点的详细状态与操作。
- **PointGrid**: 实现右侧密集矩阵，支持蛇形分布渲染、坐标轴显示及悬停提示。
- **Store**: 重建前端状态管理，整合矩阵元信息与稀疏状态逻辑。

## 风险与注意事项

- **数据兼容性**: 明确停止依赖旧表结构，历史数据可能无法自动迁移，需接受数据截断或手动处理。
- **版本控制**: 当前环境非 Git 仓库，重构过程中需手动备份以防不可逆的变更。
- **边界情况**: 动态矩阵在极小或极大点数下的 UI 表现需在实际运行中验证。
