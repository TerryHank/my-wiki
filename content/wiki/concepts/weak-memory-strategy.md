# 弱记忆策略

**弱记忆策略**是 AI 陪伴型机器人在 MVP 阶段采用的用户数据管理方案。为了降低后端开发复杂度和快速验证核心对话体验，系统刻意限制了记忆的范围和深度。

## 策略内容
- **仅保留基本信息**：系统仅记录用户的昵称等极少数静态信息。
- **无长短期记忆**：不构建复杂的用户画像、历史对话摘要或情感演变曲线。
- **会话隔离**：每次对话主要依赖当前的上下文窗口（Context Window）和预设的角色 Prompt，而非长期数据库检索。

## 优势
- **开发效率高**：避免了复杂的向量数据库、记忆检索算法和隐私数据合规问题。
- **响应速度快**：减少了云端查询记忆库的耗时，降低了端到端延迟。
- **人设稳定性**：减少了历史记忆对角色行为的干扰，使 LLM 更严格地遵循初始 Prompt 设定。

## 局限性
- **个性化不足**：机器人无法记住用户的喜好、过往经历，可能导致长期互动的重复感。
- **情感连续性弱**：无法基于过去的事件进行情感延续，表现为“金鱼记忆”。

## 未来演进
若后续需要增强陪伴感，可逐步升级为**强记忆策略**，引入向量数据库存储用户历史交互，并实现基于时间衰减的记忆检索机制。
