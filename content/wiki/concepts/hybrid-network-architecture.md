# 混合网络架构

当前开发环境采用多种网络技术并存的 **混合网络架构**，以应对复杂的设备接入和远程访问需求。

## 组成要素
- **Zero Trust 隧道**：使用 [[cloudflared]] 提供安全的远程访问，无需暴露端口。
- **虚拟组网**：使用 [[easytier]] 构建局域网或虚拟专网，连接特定 IP 设备（如 `172.20.47.98` 的机器人，`169.254.173.129` 的相机）。
- **静态 IP 直连**：部分硬件设备通过链路本地地址或固定局域网 IP 直接通信。

## 挑战与维护
- **路由冲突**：多种网络方案共存可能导致路由表冲突，需精细配置。
- **稳定性维护**：需定期通过 [[codex-cli]] 检查开机启动项（如 EasyTier）和网络连通性，解决“环境漂移”问题。
