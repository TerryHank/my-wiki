# 最小演示闭环

**最小演示闭环** (Minimum Demo Loop) 是指在硬件或嵌入式 AI 项目中，定义出的最短、最核心的用户交互路径，用于验证系统的基本可行性。它是 [[test-driven-development]] 在物理计算领域的变体，强调在复杂系统集成前，优先跑通端到端的核心功能。

## 定义与特征
- **端到端验证**：必须包含从用户输入（如触摸、语音）到系统输出（如表情、声音）的完整链路。
- **极简主义**：剔除所有非核心功能（如配网、多用户、复杂记忆），仅保留证明“系统活着”的必要逻辑。
- **快速迭代**：作为开发的首要里程碑，一旦跑通，后续功能在此基础上叠加。

## 在 AI 陪伴机器人中的应用
在 [[ai-companion-robot-mvp]] 项目中，定义了两个关键的最小演示闭环：

### 1. 触摸反馈闭环
验证设备对物理刺激的即时反应能力。
- **路径**：用户摸头 (触摸传感器) → 触发中断 → 切换 LCD 表情 (开心) → 播放预设音频。
- **目的**：验证硬件驱动、状态机逻辑及音频输出的连通性。

### 2. 语音对话闭环
验证云端智能交互链路的连通性。
- **路径**：用户说话 → 设备录音 → 上传云端 → ASR 转写 → LLM 生成回复 → TTS 合成 → 下发设备 → 播放音频 + 切换表情。
- **目的**：验证网络通信、云端 API 集成及端到端延迟的可接受度。

## 战略意义
通过聚焦于最小演示闭环，团队可以：
1. **尽早暴露风险**：如网络延迟过高、硬件驱动冲突等核心问题会在早期被发现。
2. **防止范围蔓延**：任何不能服务于跑通闭环的功能（如复杂的配网流程、多角色切换）都被推迟或砍掉。
3. **建立信心**：对于双人小团队，快速看到一个“会动、会说话”的原型是维持动力的关键。
