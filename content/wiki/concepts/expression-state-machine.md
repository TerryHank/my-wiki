# 表情状态机

**表情状态机**是 AI 陪伴型机器人设备端的核心控制逻辑，用于管理 [[lcd-eye-screen]] 的显示内容。它根据内部事件（如触摸）和外部指令（如云端返回的状态）在不同视觉状态间切换。

## 状态定义
MVP 版本至少包含以下四种基本状态：
1. **待机 (Idle)**：默认状态，通常表现为缓慢眨眼或微弱的呼吸灯效果。
2. **眨眼 (Blink)**：周期性或随机触发的瞬时动作，增加生动感。
3. **聆听 (Listening)**：当检测到用户说话或正在录音上传时显示，通常为波浪线或专注眼神。
4. **开心 (Happy)**：当检测到触摸事件或收到正面回复时显示，表现为弯月眼或特定动画。

## 触发机制
- **本地触发**：触摸传感器中断直接触发“开心”状态，无需等待云端响应，以确保反馈的即时性（<100ms）。
- **远程触发**：云端 API 返回指令，指示对话结束后的表情状态（如思考、疑惑、开心）。

## 实现要点
- **优先级管理**：本地触摸反馈通常具有最高优先级，可打断当前的聆听状态。
- **超时复位**：任何非待机状态在持续一定时间后应自动返回待机状态，防止状态卡死。
