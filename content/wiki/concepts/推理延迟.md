# 推理延迟

**推理延迟** (Inference Latency) 是衡量 AI 模型性能的核心指标之一，定义为从输入数据进入模型到模型输出结果所经历的时间间隔，通常以毫秒 (ms) 为单位。

## 重要性

- **实时性关键**：在自动驾驶、机器人控制、实时视频分析等应用中，高延迟可能导致系统反应滞后，甚至引发安全事故。
- **用户体验**：在交互式应用（如语音助手、AR 滤镜）中，超过 100ms 的延迟通常会被用户感知为“卡顿”或“不自然”。
- **流水线瓶颈**：在多阶段处理流水线中，推理延迟往往是决定整体系统吞吐量的瓶颈。

## 影响因素

1.  **硬件架构**：
    - **CPU**：通用性强但并行计算能力弱，延迟通常最高。
    - **GPU**：并行计算能力强，但在小批量或低负载下可能因调度开销导致延迟不如预期。
    - **NPU**：专为矩阵运算设计，通常在特定模型的推理上具有最低延迟。例如，[[intel-npu]] 在运行 [[yolo12]] 时延迟仅为 31.06ms，显著低于 CPU (61.91ms) 和 GPU (43.19ms)。

2.  **模型复杂度**：参数量越大、层数越深，计算延迟越高。

3.  **优化技术**：
    - **模型量化**：将浮点运算转为整数运算（如 INT8），可大幅降低延迟。
    - **算子融合**：减少内存访问次数，提升计算效率。
    - **编译优化**：使用 TensorRT, OpenVINO 等推理引擎进行图优化。

4.  **批处理大小 (Batch Size)**：
    - 增大 Batch Size 通常能提高吞吐量 (FPS)，但可能会增加单帧的平均延迟（因为需要等待凑够一批数据）。对于实时性要求高的场景，通常使用 Batch Size = 1。

## 与帧率 (FPS) 的关系

推理延迟与帧率互为倒数关系（在 Batch Size=1 时）：
$$ \text{FPS} = \frac{1000}{\text{Latency (ms)}} $$

- 延迟 31.06ms $\approx$ 32.20 FPS
- 延迟 43.19ms $\approx$ 23.16 FPS
- 延迟 61.91ms $\approx$ 16.15 FPS
