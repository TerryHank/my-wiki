1. **全栈设备工程师**
2. **AI / 后端工程师**

> 做一个**不会动的毛绒 AI 玩偶**，能 **触摸有反馈、眼睛有表情、能听你说话并回复**。

------

## 两人最小 MVP 分工表

| 角色            | 负责范围                   | 必做需求                                                     | 交付物                                       |
| --------------- | -------------------------- | ------------------------------------------------------------ | -------------------------------------------- |
| 全栈设备工程师  | 硬件 + 嵌入式 + 设备端应用 | 选主控方案；接麦克风、扬声器、LCD眼睛屏、触摸传感器；实现开机、联网、录音、播放、触摸检测、表情切换；打通设备和云端通信 | 可运行样机、固件、设备端程序、基础硬件       |
| AI / 后端工程师 | AI 对话 + 云服务           | 提供语音识别、LLM 回复、语音合成；定义角色人设；做一个最小后端接口，接收设备录音并返回文本/语音/表情状态；保存少量用户记忆 | AI 服务、接口服务、角色 Prompt、基础记忆逻辑 |

------

## MVP 只保留这 5 个功能 ✨

| 功能         | 说明                                 | 谁负责          |
| ------------ | ------------------------------------ | --------------- |
| 触摸反馈     | 摸头后切换开心表情，并播放一句回应   | 全栈设备工程师  |
| LCD 眼睛表情 | 至少有待机、眨眼、聆听、开心 4种状态 | 全栈设备工程师  |
| 语音输入     | 用户说话后，设备录音转为文字后并上传 | 全栈设备工程师  |
| AI 智能回复  | 云端返回一句符合角色设定的话         | AI / 后端工程师 |
| 语音播报     | 把 AI 回复转成语音，在设备上播出来   | 两人配合        |

------

## 两个人各自最小任务

| 人员            | 最小任务清单                                                 |
| --------------- | ------------------------------------------------------------ |
| 全栈设备工程师  | 做出一台能开机的玩偶原型；接通屏幕、麦克风、喇叭、触摸；实现“摸一下有反应、说一句能上传、收到结果能播出来” |
| AI / 后端工程师 | 做一个最简单服务：语音转文字 → 大模型生成回复 → 文字转语音 → 返回给设备；同时写好固定角色人设 |

## 最小软件架构

| 部分     | 最简做法                                 |
| -------- | ---------------------------------------- |
| 设备端   | 录音、上传、播放、切表情、读触摸         |
| 云端     | 一个 API 接口                            |
| AI       | 直接接现成 ASR + LLM + TTS               |
| 用户系统 | 先不要                                   |
| 记忆系统 | 先只记住用户昵称                         |
| 配网     | 先用最简单方式，甚至开发阶段先写死 Wi-Fi |

------

## 最小演示闭环

**用户摸头** → **眼睛变开心** → **玩偶说“嘿嘿，你在摸我呀”**
**用户说话** → **设备录音上传** → **AI 生成角色回复** → **设备播报并切换表情**
