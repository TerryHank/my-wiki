# 验收表单交互系统实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 构建一个基于 VitePress + Node.js + SQLite 的单机内网验收录入系统，支持 `1200` 点位、`3` 个场景、每场景 `5` 次点击录入、右侧二维点阵总览和自动保存。

**架构：** 使用 `docs/` 作为 VitePress 站点目录，业务 UI 通过 Vue 组件承载；使用 Node.js 原生 HTTP 服务提供静态文件和 REST API；使用 SQLite 持久化固定单表数据；前后端共享点位和场景常量，避免映射不一致。

**技术栈：** VitePress、Vue 3、Node.js、SQLite（better-sqlite3）、Vitest、TypeScript

```ts
export const GRID_COLUMNS = 40
export const GRID_ROWS = 30
export const TOTAL_POINTS = GRID_COLUMNS * GRID_ROWS

export function getPointPosition(pointId: number) {
  const zeroBased = pointId - 1
  return {
    row: Math.floor(zeroBased / GRID_COLUMNS) + 1,
    col: (zeroBased % GRID_COLUMNS) + 1
  }
}

export function getPointIdFromPosition(row: number, col: number) {
  return (row - 1) * GRID_COLUMNS + col
}
```

```ts
export function getPointStatus(results: ResultRecord[]): PointStatus {
  const checkedCount = results.filter((item) => item.checked).length
  if (checkedCount === 0) return 'not-started'
  if (checkedCount === 15) return 'completed'

  const sceneCount = new Set(results.filter((item) => item.checked).map((item) => item.scene)).size
  return sceneCount === 3 ? 'in-progress-advanced' : 'in-progress'
}
```
