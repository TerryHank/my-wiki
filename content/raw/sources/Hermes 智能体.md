# 知识库 — Vault 规范

## 目录结构
- `/原始资料` — 源材料、剪藏文章、研究资料（输入区）
- `/知识库` — LLM 编译的知识库（见下方 Wiki 系统）
- `/输出` — 查询结果和生成的报告

## Wiki 系统
你是 `知识库/` 文件夹的管理员。所有内容由你编写和维护。

### 结构
- `知识库/总索引.md` 是入口——列出每个主题及一行描述。
- 每个主题有自己的子文件夹，内含 `_索引.md` 列出该主题下所有文章。

### 编译（compile）
当我说"编译"或在 `原始资料/` 中投放新材料时：
1. 读取每个原始文件
2. 判断它属于哪个主题（或创建新的主题文件夹）
3. 编写知识库文章，包含核心要点和 `[[双链]]` 关联概念
4. 更新该主题的 `_索引.md`
5. 更新 `知识库/总索引.md`
6. 如果原始文件跨多个主题，在两个主题中都创建文章并交叉链接

### 查询（query）
当回答知识库相关问题时：
1. 先读 `知识库/总索引.md` 找到正确主题
2. 读该主题的 `_索引.md` 找到相关文章
3. 读具体文章
4. 综合回答

### 审计（audit）
当我说"审计"或"检查"时，审查知识库：
- 不一致或矛盾的信息
- 缺失的交叉链接
- 覆盖空白
- 提出改进建议，但不确认不做修改

## 规范
- 引用其他笔记时始终使用 `[[双链]]`
- 文件名：中文为主，保留关键英文术语
- 文章保持简洁——要点列表优于长段落
- 每篇知识库文章必须包含 `## 核心要点` 章节
- Frontmatter 必须包含：title, created, updated, type, tags, sources
