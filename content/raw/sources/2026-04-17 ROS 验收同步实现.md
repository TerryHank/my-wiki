# ROS 验收联动实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 让 `192.168.6.99` 上的 ROS 工作区把区域进度、全量规划点阵和当前组举证同步到本前端，并自动记录绑扎进度。

**架构：** 在 `suoquNode.cpp` 中新增一个 `std_msgs/String` 规划话题 `/acceptance/plan`，把 `pseudo_slam_bind_path.json` 的内容作为前端专用 JSON 发布。前端新增 `acceptance-plan` 订阅模块，收到规划后自动重建点阵和行列。现有 `/cabin/area_progress` 继续负责自动跳组，现有 `/coordinate_point` 与 `/pointAI/result_image` 负责当前组举证，前端在区域完成事件到达时调用本地 Node API 自动落库。

**技术栈：** ROS1 Noetic、C++ `roscpp`、`std_msgs/String`、Vue 3、VitePress、Node.js、SQLite、Vitest。

```cpp
#include <std_msgs/String.h>

ros::Publisher pub_acceptance_plan;

void publish_acceptance_plan_json(const std::string& json_path)
{
    std::ifstream file_obj(json_path);
    if (!file_obj.is_open()) {
        return;
    }

    std::stringstream buffer;
    buffer << file_obj.rdbuf();
    std_msgs::String msg;
    msg.data = buffer.str();
    pub_acceptance_plan.publish(msg);
}
```

- [ ] **步骤 3：实现 `acceptance-plan.ts`**

```ts
export function parseAcceptancePlanJson(raw: string): AcceptancePlanState | null {
  const parsed = JSON.parse(raw) as RosAcceptancePlanPayload
  const points = parsed.areas.flatMap((area) =>
    area.groups.flatMap((group) =>
      group.points.map((point) => ({
        areaIndex: area.area_index,
        groupIndex: group.group_index,
        globalIdx: point.global_idx,
        globalRow: point.global_row,
        globalCol: point.global_col
      }))
    )
  )
  return buildAcceptancePlanState(points)
}
```
